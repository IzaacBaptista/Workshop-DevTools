🐛 Debug Playground

Ambiente de prática para aprender a usar o DevTools — Network, Console e Application.

Cada card abaixo simula um problema real. Clique em ▶ Reproduzir, observe o que aparece no DevTools e tente diagnosticar o que aconteceu antes de clicar em 👁 Ver resposta.

🧪 Desafios de Debug

❌

Cadastro não salva

500

O botão de cadastrar parece funcionar, mas o usuário nunca é salvo. Abra a aba Network e clique em Reproduzir.

🔍 O que aconteceu?
  • O servidor retornou HTTP 500 Internal Server Error.
  • Onde olhar: aba Network → clique na requisição → aba Response.
  • O corpo da resposta traz a mensagem de erro do servidor.
📝 Exemplo de bom bug report:
  • Ação: clicar em "Salvar cadastro"
  • Request: POST /api/users
  • Status: 500
  • Response: {"error":"Erro interno ao salvar cliente"}
⚠️

Campo obrigatório ignorado

422

O formulário envia sem e-mail. O servidor rejeita a requisição. Abra Network, veja o payload e a resposta.

🔍 O que aconteceu?
  • O campo email foi enviado vazio.
  • O servidor retornou HTTP 422 Unprocessable Entity.
  • Onde olhar: Network → Payload (corpo enviado) e Response.
📝 Exemplo de bom bug report:
  • Request: POST /api/users/validate
  • Payload enviado: {"name":"João","email":""}
  • Status: 422
  • Response: {"error":"Email is required"}
🔐

Usuário foi deslogado

401

A tela de perfil não carrega. O token de sessão expirou. Observe o header de autenticação na aba Network.

🔍 O que aconteceu?
  • O token enviado no header Authorization está expirado/inválido.
  • O servidor retornou HTTP 401 Unauthorized.
  • Onde olhar: Network → Headers (veja o token enviado) → Response.
📝 Bom bug report:
  • Request: GET /api/profile
  • Header: Authorization: Bearer token-expirado
  • Status: 401
  • Response: {"message":"Sessão expirada. Faça login novamente."}
🚫

Ação bloqueada pelo sistema

403

O usuário tenta executar uma ação de administrador sem permissão. Veja o status code na aba Network.

🔍 O que aconteceu?
  • O servidor reconheceu o usuário (diferente do 401), mas negou a ação.
  • Retornou HTTP 403 Forbidden.
  • Diferença chave: 401 = não autenticado · 403 = sem permissão.
📝 Bom bug report:
  • Request: DELETE /api/admin/action
  • Status: 403
  • Response: {"message":"Você não tem permissão para realizar esta ação."}
💥

Tela quebra ao clicar

JS Error

O botão dispara um erro JavaScript antes de chamar qualquer API. Abra a aba Console — não a Network!

🔍 O que aconteceu?
  • Um TypeError: Cannot read properties of undefined foi lançado no JavaScript.
  • Nenhuma requisição de rede foi feita — olhar na Network não ajuda aqui.
  • Onde olhar: aba Console (erros em vermelho).
📝 Bom bug report:
  • Console mostra: TypeError: Cannot read properties of undefined (reading 'nome')
  • Nenhuma requisição aparece na Network.
  • O erro ocorre ao clicar no botão "Reproduzir" do Desafio 5.
🐢

Sistema muito lento

Timing

A página demora para carregar os dados. Observe o tempo na coluna Time da aba Network (aguarde ~8 segundos).

🔍 O que aconteceu?
  • O servidor demorou ~8 segundos para responder.
  • Onde olhar: Network → coluna Time (waterfall).
  • Dica extra: use a throttling do DevTools para simular conexões lentas.
📝 Bom bug report:
  • Request: GET /api/slow
  • Status: 200 (mas tardio)
  • Tempo de resposta: ~8 000 ms
  • Captura de tela do waterfall anexada.
🍪

Sessão inválida (cookie)

Cookie

O sistema perde a sessão do usuário. Verifique a aba Application → Cookies e depois clique em Reproduzir.

🔍 O que aconteceu?
  • O cookie session está ausente ou com valor inválido.
  • Onde olhar: aba Application → Storage → Cookies.
  • O servidor retornou 401 indicando sessão inválida.
📝 Bom bug report:
  • Comportamento: usuário aparece como deslogado sem ter feito logout.
  • Cookie session ausente na aba Application.
  • Request: GET /api/session-check → Status: 401
📦

Dados estranhos salvos

localStorage

O app salva preferências corrompidas no localStorage. O comportamento fica estranho. Abra Application → Local Storage.

🔍 O que aconteceu?
  • O app salvou um valor inválido em localStorage.userPrefs.
  • Onde olhar: Application → Storage → Local Storage.
  • Para corrigir: selecione a chave userPrefs e delete-a.
📝 Bom bug report:
  • Comportamento: tema do app não carrega corretamente.
  • Local Storage → userPrefs: {"theme":"undefined","lang":null}
  • Corrección: limpar a chave no DevTools resolve imediatamente.
⚠️

Aviso no console (não quebra)

Warning

O app ainda funciona, mas o Console mostra um warning. Aprenda a diferenciar warning de erro.

🔍 O que aconteceu?
  • A API retornou deprecated_flag: true — o frontend emitiu um console.warn.
  • O app continua funcionando normalmente (não é um erro crítico).
  • Onde olhar: aba Console → mensagens em amarelo = warnings.
📝 Como diferenciar:
  • Erro (vermelho): quebra o fluxo, precisa de ação imediata.
  • Warning (amarelo): algo a melhorar, mas não bloqueia o usuário.
  • Info (azul): log informativo do sistema.
